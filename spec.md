# 攀岩日记小程序

## 目标与范围

### 产品目标
做一个轻量的攀岩记录小程序，让用户能快速记录每次攀岩的：
- 日期（必填）
- 攀岩馆（必填，支持高德地图 POI 模糊搜索选择）
- 心得（可选，文本）
并提供记录列表、筛选、攀岩日历高亮与基础统计（可选）。

### 核心原则
- 记录必须“快”：创建流程 ≤ 30 秒完成
- 地点选择必须“准”：基于 POI 的模糊搜索 + 可选定位附近排序
- 离线优先：无网络也能查看/新增（POI 搜索需要网络）
- 隐私优先：定位非强制；明确告知用途；不做持续定位

### 不在本期范围（可后续迭代）
- 社交、公开动态、关注
- 训练计划、线路难度体系、成绩榜
- 登录/云同步/多端一致（除非后续改造）

---

## 技术方案概览（MVP）

### 架构原则
- 本地优先：记录数据只落本地；无网络不影响查看/新增/编辑/删除
- 在线增强：仅“场馆搜索/定位附近”依赖网络
- 安全优先：高德 Key 不暴露在客户端；统一由后端代理请求

### 模块划分（建议）
- `storage`：本地读写、schema 版本、迁移
- `records`：Record CRUD、派生数据（按日分组/计数）
- `poi`：POI 搜索（调用后端代理）、POI 模型归一化
- `ui`：列表/日历/新建编辑/搜索页

## 用户与使用场景

### 目标用户
- 普通攀岩爱好者：每次攀完记录一下日期/场馆/感受
- 复盘型用户：希望按场馆/时间快速回看心得与频次

### 典型场景
- 攀完下墙：打开小程序 → 新建 → 搜索场馆 → 保存
- 想复盘某馆体验：筛选“某攀岩馆” → 查看过往心得
- 想看攀岩频率：打开“日历”看当月高亮日期

## 关键交互（MVP）

### 新建记录流程
- 进入新建页 → 默认日期=今天
- 点击“选择攀岩馆” → 场馆搜索页
- 输入关键词（如“攀岩/岩/Climb/商场名”）→ 列表结果
- 选中某条 POI → 返回新建页并填入场馆
- 心得可不填 → 保存成功 → 返回列表（顶部刷新）

### 场馆模糊搜索体验要求
- 输入变化触发防抖（建议 300ms）
- 建议 2 个字符起搜（可配置）
- 空态提示：示例关键词
- 无结果：建议更换关键词/尝试定位附近
- 失败：展示可重试，并允许“手动输入场馆名”兜底

### 删除与编辑
- 删除：二次确认；删除后列表与日历高亮即时更新
- 编辑：修改日期/场馆/心得后，列表与日历高亮即时更新

## 数据模型（纯本地）

### Record（攀岩记录）
- id: string（UUID）
- date: string（ISO `yyyy-mm-dd`，以用户本地时区为准）
- gym:
  - poiId: string | ""（高德 POI `id`；手动输入兜底时可空）
  - name: string（必填）
  - address: string（可选）
  - location:
    - lat: number | null
    - lng: number | null
  - city: string（可选）
  - provider: "amap" | "manual"
- note: string（可选，0..2000）
- createdAt: number（ms timestamp）
- updatedAt: number（ms timestamp）
- deletedAt: number | null（MVP 可不做软删）

### Gym（场馆/POI 归一化模型）
- poiId: string | ""（高德 POI `id`；手动输入兜底时可空）
- name: string
- address: string
- location:
  - lat: number | null
  - lng: number | null
- city: string
- provider: "amap" | "manual"

### 示例 JSON（Record）

```json
{
  "id": "5c10f9e2-5a28-4d08-86f6-3b0af5f7d3f8",
  "date": "2026-04-07",
  "gym": {
    "poiId": "B0FFG8XXXX",
    "name": "XX攀岩馆",
    "address": "XX路XX号",
    "location": { "lat": 31.230416, "lng": 121.473701 },
    "city": "上海市",
    "provider": "amap"
  },
  "note": "今天刷了几条抱石，体感不错。",
  "createdAt": 1775520000000,
  "updatedAt": 1775520000000,
  "deletedAt": null
}
```

### 派生/缓存（建议）
- recentGyms: gym[]（最多 10）
- dateSet: `{ [yyyy-mm-dd]: number }`（当日记录数，用于日历高亮/角标）
- recordsByDate: `{ [yyyy-mm-dd]: Record[] }`（用于点选某天快速展示）

### 存储策略
- MVP：本地存储（小程序 Storage）
- 版本：`schemaVersion: number`，用于未来迁移

### schemaVersion 与迁移（建议）
- `schemaVersion = 1`：仅 `records` + `schemaVersion`
- 迁移触发：应用启动时读取 `schemaVersion`，小于当前版本则执行迁移并写回
- 迁移策略：
  - 只做向前兼容：缺失字段补默认值（如 `deletedAt=null`、`gym.provider="manual"`）
  - 严禁破坏性迁移：不丢数据；新增字段要可回填

## 地图与POI模糊搜索（高德地图）

### Provider
- 高德地图 Web 服务 API（AMap Web Service）
- 使用能力：输入提示（Autocomplete/Inputtips）+ POI 关键字搜索（Place Text）

> 安全建议：不在客户端直接暴露高德 Key。推荐走你自己的后端代理（或云函数）统一签名、限流与配额控制。

### 后端代理（推荐做法，MVP）
目标：客户端只请求你自己的域名，服务端再去请求高德，避免 Key 泄露与被盗刷。

#### 约束
- 不需要用户登录
- 允许匿名调用，但必须做限流（IP/设备维度）与最小化字段透传
- 记录日志只用于排障与风控，不存用户精确位置历史

#### 代理接口（建议契约）
1) 输入提示
- `GET /api/poi/inputtips?keywords=...&city=...`
- 响应：统一返回 `tips[]`（仅保留展示所需字段）

2) POI 检索
- `GET /api/poi/search?keywords=...&city=...&page=1&pageSize=20`
- 响应：统一返回 `pois[]`（用于落库）

3)（可选）逆地理/详情
- `GET /api/poi/detail?id=...`
- 响应：统一返回一个 `Gym`

#### 错误码（建议）
- `400`：参数缺失（如 keywords 为空或过短）
- `429`：触发限流
- `502`：上游高德失败/超时
- `200` 但返回空数组：无结果（前端做空态）

#### 响应结构示例（建议）
1) `GET /api/poi/inputtips`

```json
{
  "tips": [
    { "name": "XX攀岩馆", "district": "浦东新区", "address": "XX路XX号", "location": "121.473701,31.230416", "id": "B0FFG8XXXX" }
  ]
}
```

2) `GET /api/poi/search`

```json
{
  "pois": [
    {
      "poiId": "B0FFG8XXXX",
      "name": "XX攀岩馆",
      "address": "XX路XX号",
      "location": { "lng": 121.473701, "lat": 31.230416 },
      "city": "上海市",
      "provider": "amap"
    }
  ],
  "page": 1,
  "pageSize": 20,
  "hasMore": false
}
```

### 接口与流程（推荐）
#### 阶段 1：输入提示（联想）
- 接口：`/v3/assistant/inputtips`
- 入参：
  - key（由服务端代理注入）
  - keywords：用户输入
  - city（可选）：当前城市或用户选择城市
  - citylimit（可选）：是否限制城市
- 输出：tips 列表（用于联想展示）

#### 阶段 2：POI检索（可落库）
- 接口：`/v3/place/text`
- 入参：
  - key（由服务端代理注入）
  - keywords：关键词
  - city（可选）
  - page / offset（分页）
  - extensions=all（尽量拿全字段）
- 输出：pois 列表（含 id/name/address/location 等）

### POI 字段映射（落到 Record.gym）
- `poiId`: 高德 `id`
- `name`: 高德 `name`
- `address`: 高德 `address`（或 `district + address` 组合展示）
- `location`: 高德 `location` 字符串 `"lng,lat"` 拆分为数值
- `city`: `cityname`（可选）
- `provider`: 固定 `"amap"`

### 结果去重与展示排序（建议）
- 去重：以 `poiId` 为主键；若 `poiId` 缺失则以 `name+address` 做弱去重
- 排序（无定位）：按高德默认相关性顺序
- 排序（有定位，可选）：计算距离后“距离优先 + 相关性兜底”

### 手动场馆与 POI 场馆的合并策略（建议）
- 列表/筛选/统计时的场馆主键：
  - 优先使用 `provider=amap` 的 `poiId`
  - `provider=manual` 使用规范化后的 `name`（去空格、全半角统一）作为弱主键
- 编辑记录时允许将 `manual` 场馆“补齐为 POI”，补齐后应视为新主键（后续统计按 POI 归集）

### 定位能力（可选增强）
- 定位不是必需；未授权也能关键词搜索
- 若授权定位：
  - 用途：附近优先排序/距离展示/附近检索（可选）
  - 不做持续定位，不保存轨迹
- 定位失败不影响关键词搜索

### 失败与降级
- POI 服务不可用/无结果：
  - 显示错误或空态
  - 提供“手动输入场馆名”兜底（provider=manual）

## 新增页面：攀岩日历（Calendar）

### 目标
以“月视图日历”展示攀岩日期；**有记录的日期高亮**；点击某日可查看当天记录并支持一键新建。

### 页面结构
- 顶部：月份标题 + 上月/下月按钮 + “回到本月”
- 中部：月视图网格（日/一/二/三/四/五/六）
- 底部：选中日期的记录列表（无记录则空态 + 新建按钮）

### 高亮与展示规则
- 若某日 `dateSet[date] >= 1`：该日期显示高亮
- 若某日记录数 > 1：日期角标显示数量（可选，9+）
- 今日样式与高亮可叠加（需定义优先级与视觉规范）
- 选中日样式：与高亮区分（例如描边/背景不同）

### 交互
- 点击某日：
  - 若有记录：展示当天记录列表（按 createdAt 倒序）
  - 若无记录：提示“当天暂无记录”，提供“新建并带入日期”
- 月份切换：重新计算当月网格与 highlightDates
- 日历页进入时默认选中“今天”

### 数据计算（纯本地）
- 输入：records
- 派生：
  - highlightDates：当月内有记录的日期集合
  - recordsByDate：按 `Record.date` 分组
  - dateSet：按 `Record.date` 计数
- 性能：
  - 月切换计算应轻量（建议使用缓存或增量更新）

### 边界条件
- 日期统一存 `yyyy-mm-dd`，避免时区偏移导致跨日
- 删除当天最后一条记录后：高亮应消失
- 编辑记录日期跨天：两个日期的高亮与计数都应正确更新

### 验收标准（MVP）
- 新建记录后，对应日期在日历中立即高亮
- 点击高亮日期可看到当天记录列表并进入详情
- 删除当天最后一条记录后，该日期高亮消失
- 切换月份高亮准确

## 校验规则

### 新建/编辑校验
- date 必填，格式合法（`yyyy-mm-dd`）
- gym 必填：至少 `name` 必须有；优先要求 `poiId`（若选择 POI），手动兜底允许 `poiId=""`
- note 可空，长度 0..2000

### 删除策略
- MVP：硬删除
- 可选：软删（deletedAt）以支持恢复与统计一致性

## 列表与筛选

### 列表排序
- 默认按 date 倒序
- 同日多条按 createdAt 倒序

### 筛选项（MVP）
- 日期范围：最近7天/30天/本月/自定义
- 场馆：从已记录场馆中选择（按 poiId 优先去重；manual 以 name 去重）

### 搜索（可选）
- 关键词匹配：场馆名、地址、心得

## 统计口径（可选）

- 总次数：记录条数
- 本月次数：date 落在本月
- Top 场馆：按 `gym.poiId`（优先）或 `gym.name`（manual 兜底）分组计数
- 最近一次攀岩距今天数：today - max(date)

## 非功能需求

### 性能
- 记录列表 1000 条以内流畅滚动
- 日历切换月份不卡顿（计算/渲染优化）

### 可用性
- 离线可用：查看/新增/编辑本地记录均可
- 异常提示清晰：网络错误、权限未开、无结果

### 隐私与合规
- 定位非必需、默认不请求
- 仅在用户点击“附近/定位”时请求授权
- 不上传轨迹与精确定位历史

### 可维护性
- POI provider 抽象：`searchPoi(keyword, city, location?)` 返回统一模型
- schemaVersion 支持未来迁移（如加字段、加软删、加同步）

### 视觉与交互一致性（建议）
- 日历高亮：使用主色浅底/圆角底纹；选中日使用描边；今日使用小圆点或角标
- 列表项信息密度：一屏至少可见 6 条记录（常规手机）
- 空态文案：
  - 列表空态：引导“去新增第一条攀岩记录”
  - 日历选中日无记录：引导“一键新建（日期已填）”
  - POI 无结果：提示换关键词 + 手动输入兜底

## 测试计划（MVP）

### 功能测试
- 新建/编辑/删除记录
- 场馆搜索：中文/英文/混输、无结果、网络断开、重试
- 手动场馆兜底保存与后续编辑补齐 POI
- 筛选：日期范围 + 场馆组合
- 日历：高亮正确、点选显示当天记录、删除后高亮更新、跨月切换正确

### 兼容性与异常
- iOS/Android 主流机型
- 弱网/断网（POI 搜索失败提示；本地功能不受影响）

### 回归用例（建议最小集）
- 新建：选择 POI -> 保存 -> 列表与日历同时反映
- 新建：手动输入场馆 -> 保存 -> 筛选按场馆可选中（manual）
- 编辑：修改日期到另一日 -> 两个日期高亮与计数正确更新
- 删除：删除某日最后一条 -> 该日高亮消失
- 搜索：输入过短/空 -> 不请求上游，提示完善关键词
- 限流：触发 429 -> 展示“稍后再试”
